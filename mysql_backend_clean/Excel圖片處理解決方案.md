# Excel 圖片處理解決方案

## 問題摘要
原先的問題是 Excel 檔案中的圖片無法正確儲存到 MySQL 資料庫中。Excel 儲存格中顯示的「圖片」文字僅是佔位符，實際的圖片是漂浮在儲存格上方的物件（繪圖物件/形狀）。

## 解決方案
我們建立了強化版的 Excel 處理器，能夠正確擷取並儲存這些圖片：

1. 將 Excel 檔案視為 ZIP 壓縮檔，以存取儲存在 `xl/media/` 中的原始媒體檔案
2. 使用 openpyxl 直接存取 Excel 中的繪圖物件（浮動圖片）
3. 將擷取的圖片轉換為二進位格式，以便儲存在 MEDIUMBLOB 資料欄位中

## 關鍵發現
- Excel 圖片實際上並非儲存在儲存格內 —「圖片」文字僅是佔位符
- 真正的圖片是漂浮在工作表格線上方的浮動元素（繪圖物件/形狀）
- 這些圖片儲存在 Excel 檔案內部的 ZIP 結構中，位於 `xl/media/` 資料夾
- 標準的 Excel 讀取函式庫如 pandas 無法擷取這些浮動圖片
- 我們需要專門的處理方法來擷取 Excel 檔案結構中的這些圖片

## 主要功能檔案
1. `excel_handler.py` - 強化版 Excel 處理器，具備完整的圖片擷取功能
2. `db_connection.py` - 資料庫連接和操作相關功能
3. `api.py` - API 功能實作
4. `app.py` - FastAPI 應用程式入口

## 實作細節

### 1. 圖片擷取方法
我們實作了兩種從 Excel 擷取圖片的方法：

- **ZIP 壓縮檔方法**：將 Excel 檔案視為 ZIP 壓縮檔，並從 `xl/media/` 擷取圖片檔
- **openpyxl 方法**：使用 openpyxl 載入工作簿並直接存取其繪圖物件

### 2. 與現有程式碼的整合
強化版解決方案保留了所有現有功能，同時新增了適當的圖片處理：

- 維持所有現有的 Excel 解析邏輯
- 解決方案現在包含專門的圖片擷取方法
- 圖片會自動與問題建立關聯
- 保留原始的儲存格圖片檢查作為備用方法

### 3. 資料庫儲存
此解決方案利用先前的資料庫改進：

- 使用 MEDIUMBLOB 資料欄位類型儲存二進位圖片資料
- 正確處理二進位資料的插入和擷取
- 在整個處理過程中維持資料完整性

## 設定需求

2. 資料庫有 `picture` 欄位設為 MEDIUMBLOB 類型（已在先前步驟完成）
